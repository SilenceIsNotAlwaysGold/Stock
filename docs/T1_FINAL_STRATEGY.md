## 最终策略设计

### 选股逻辑（G12验证有效）

基于你的历史回测，G12配置是唯一验证有效的：

```python
1. 上影线 < 1%          # 收盘接近最高价
2. 前日涨幅 < 3%        # 非追涨
3. 连续上涨 ≤ 2天       # 非连续大涨
```

**历史验证数据**：
- 样本：500股
- 胜率：66.7%
- 收益：+26.54%
- 最大回撤：-4.36%

### 卖出逻辑（优化止盈止损）

基于v5/v6/v7的教训，优化卖出时机：

```python
优先级1: 开盘止损-5%      # 放宽容忍度，减少误伤
优先级2: 开盘盈利≥0.8%    # 快速止盈
优先级3: 盘中冲高≥1.5%    # 降低阈值，及时兑现
优先级4: 9:35固定卖出     # 早卖避免回落
```

**关键改进**：
- 取消-2%盘中止损（v5失败根源）
- 放宽开盘止损至-5%（v6的-3%仍偏紧）
- 降低止盈阈值（0.8%/1.5% vs v6的2%/3%）
- 提前卖出时间（9:35 vs v6的9:45）

## 核心经验教训

### 1. 技术指标不可靠
- RSI/KDJ在A股T+1环境下失效
- 低位≠反弹，超卖可以更超卖

### 2. 简单即有效
- G12三因子胜率66.7%
- 复杂的v4/v5/v6/v7都不如它

### 3. 止损设置至关重要
- v5的-2%盘中止损毁掉30%交易
- v6的vwap_exit胜率66.1%，说明选股没问题，是止损太敏感

### 4. 快速止盈优于等待
- v6中auction_sell和intraday_high胜率100%
- 但fixed_time_945占83.6%，胜率只有45.26%
- 结论：早卖比晚卖好
