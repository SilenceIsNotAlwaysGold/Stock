## 1. 概述

### 1.1 一句话描述

全面升级 T+1 隔夜策略（尾盘买入、次日早盘卖出），新增大盘环境、资金流向、市场情绪、板块热度四大过滤维度，重构早盘智能卖出引擎，目标胜率 60%+。

### 1.2 核心价值

现有 T1 策略存在四大问题：选股太少、选出来不赚钱、缺少关键过滤维度、卖出时机差。v2 版本通过多维度过滤提升选股质量，通过智能卖出提升盈利能力，最终实现稳健的隔夜收益。

### 1.3 目标用户

- **主要用户**: 使用本平台进行 A 股量化交易的个人投资者
- **策略风格**: 高胜率稳健型（宁可少做，每次做要有 60%+ 胜率，单次收益 1-3%）

### 2.2 验收标准

#### R-001: 大盘环境过滤层
- **WHEN** 上证指数收盘价低于 MA20 且 MACD 空头运行, **THEN** 系统 **SHALL** 自动跳过当日 T1 选股扫描
- **WHEN** 市场涨跌比 < 0.8（下跌家数多于上涨家数 25%+）, **THEN** 系统 **SHALL** 标记为"弱势市场"并提高选股阈值
- **WHEN** 大盘环境评分 ≥ 60 分（满分 100）, **THEN** 系统 **SHALL** 正常执行 T1 扫描

#### R-002: 资金流向确认
- **WHEN** 候选股当日主力净流入为负, **THEN** 系统 **SHALL** 排除该候选股
- **WHEN** 候选股当日主力净流入 > 500 万且占成交额 > 3%, **THEN** 系统 **SHALL** 给予加分

#### R-003: 市场情绪指标
- **WHEN** 全市场涨停数 > 80 且涨跌比 > 2.0, **THEN** 系统 **SHALL** 适度放宽选股条件（降低量比阈值 10%）
- **WHEN** 全市场跌停数 > 50 或涨跌比 < 0.5, **THEN** 系统 **SHALL** 暂停 T1 选股

#### R-004: 板块热度过滤
- **WHEN** 候选股所属板块当日涨幅排名不在前 20%, **THEN** 系统 **SHALL** 排除该候选股
- **WHEN** 候选股所属板块有 3 只以上涨停, **THEN** 系统 **SHALL** 给予板块热度加分

#### R-005: 早盘智能卖出引擎
- **WHEN** 开盘价 ≥ 买入价 * 1.05, **THEN** 系统 **SHALL** 集合竞价直接卖出（止盈）
- **WHEN** 开盘后 15 分钟内股价跌破买入价 * 0.97, **THEN** 系统 **SHALL** 立即止损卖出
- **WHEN** 开盘后 15 分钟股价在买入价 ±3% 内震荡, **THEN** 系统 **SHALL** 继续观察至 30 分钟
- **WHEN** 30 分钟后股价仍未达到止盈/止损线, **THEN** 系统 **SHALL** 以当前价卖出
- **WHEN** 10:30 仍有未卖出持仓, **THEN** 系统 **SHALL** 强制清仓（最终兜底）

#### R-006: 选股范围限制
- **WHEN** 股票代码以 688/8/4 开头（科创板/北交所）, **THEN** 系统 **SHALL** 排除
- **WHEN** 股票名称包含 ST/\*ST, **THEN** 系统 **SHALL** 排除
- **WHEN** 股票上市不足 60 个交易日, **THEN** 系统 **SHALL** 排除（次新股）

#### R-007: 仓位管理
- **WHEN** 当日已买入 5 只, **THEN** 系统 **SHALL** 停止继续买入
- **WHEN** 候选股评分排名, **THEN** 系统 **SHALL** 按评分从高到低选取前 3-5 只

#### R-008: 子策略独立回测
- **WHEN** 用户指定回测时间范围和子策略, **THEN** 系统 **SHALL** 输出胜率、总收益、最大回撤、夏普比率
- **WHEN** 回测完成, **THEN** 系统 **SHALL** 生成每日交易明细和收益曲线数据

## 4. 技术约束

### 4.1 技术栈

- **后端**: Python 3.11+ / FastAPI / SQLAlchemy (async)
- **策略引擎**: 继承现有 `engine/base.py` 的 `BaseStrategy` 体系
- **数据源**: Tushare（主）/ AKShare（备）/ BaoStock（兜底），通过 `DataSourceManager` 统一调用
- **数据库**: PostgreSQL（结构化数据）、Redis（缓存）
- **LLM**: 不涉及（纯量化策略，不依赖 AI 分析）

### 4.2 集成点

- **数据层**: `dataflows/source_manager.py` — 获取日线、资金流向、板块数据
- **策略注册**: `engine/registry.py` — 新策略需通过 `@StrategyRegistry.register` 注册
- **服务层**: `app/services/t1_service.py` — 扫描、买入、卖出、统计的核心业务逻辑
- **路由层**: `app/routers/t1_strategy.py` — API 接口
- **定时任务**: `app/routers/scheduler.py` — 尾盘扫描和早盘卖出的定时触发

### 4.3 关键设计约束

- 新增的过滤层（大盘/资金/情绪/板块）应作为独立模块，可插拔，不侵入现有子策略逻辑
- 回测引擎需支持"无数据库"模式，纯内存运行，方便本地调试
- 所有阈值参数应可配置（通过 `system_configs` 表或环境变量）
