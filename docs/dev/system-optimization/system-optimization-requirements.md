# 需求文档: A股量化系统实战优化

## 1. 概述

**一句话描述:** 将现有 T1 隔夜量化交易系统优化为适配用户真实交易权限（仅沪深主板+中小板）、小资金激进风格、全自动化运行的实战盈利系统。

**核心价值:** 让一个投资新手能通过系统每天自动选出 1-2 只高确定性的主板/中小板股票，早盘快速下单，无需盯盘，追求月收益 5%+。

**目标用户:** 小资金（<10万）、无盯盘时间、仅有沪深主板+中小板权限、偏好激进集中持仓的 A 股个人投资者。

## 2. 需求与用户故事

### 需求 R1: 交易权限硬过滤（P0 - 必须）

**用户故事:** 作为一个没有创业板/科创板/北交所/ST 权限的投资者，我需要系统绝不推荐这些股票，避免选出无法买入的标的。

**当前问题:**
- `veto_filter.py` 过滤了 688xxx（科创板）、8xxx/4xxx（北交所）、ST 股
- **创业板（300xxx）完全没有被过滤** — 这是关键缺陷
- 需要新增 300xxx 过滤规则

**验收标准:**
- [ ] 300xxx 开头的股票被 veto_filter 拒绝，原因标注"创业板（300xxx）"
- [ ] 系统扫描结果中不再出现 300/688/8/4/ST 开头的任何股票
- [ ] 现有 60xxxx（沪主板）、00xxxx/002xxx（深主板+中小板）的股票正常通过
- [ ] 过滤规则可通过配置开关控制（预留未来开通权限时关闭）

### 需求 R2: 评分模型调优 — 适配激进小资金风格（P0 - 必须）

**用户故事:** 作为小资金激进投资者，我需要系统优先选出短期爆发力强、资金驱动明显的主力股，而非稳健大白马。

**当前问题:**
- 评分权重：技术 30 + 资金 25 + 基本面 15 + 板块 15 + 大盘 15 = 100
- 基本面权重 15 分偏高（ROE/PE 对短线 T+1 影响小）
- 资金面和板块面对短线交易最关键，但权重不够突出
- TOP_N = 5 只，用户要 1-2 只集中出击

**调优方向:**
- 降低基本面权重（15→8），提升资金面权重（25→30）和板块面权重（15→17）
- 技术面增加动量因子权重、降低 K 线形态权重
- TOP_N 改为 2（每日最多推 2 只）
- min_total_score 适当上调（50→55），提高选股质量
- 动量评分的最优区间从 [2%,5%] 调整为 [1.5%,5%]（更适合主板 10% 涨跌停）

**验收标准:**
- [ ] 评分权重调整为：技术 30 + 资金 30 + 基本面 8 + 板块 17 + 大盘 15 = 100
- [ ] TOP_N 默认改为 2
- [ ] min_total_score 上调至 55
- [ ] 回测对比调优前后的胜率和收益率差异

### 需求 R3: 卖出引擎参数适配激进风格（P1 - 重要）

**用户故事:** 作为追求高收益的投资者，我不希望赚 3% 就跑，也不希望亏 2% 就割，要给强势股更多空间。

**当前参数问题:**
- phase1_take_profit = 5%（高开止盈 — 合理）
- phase2_take_profit = 3%（盘中止盈 — 对强势股偏保守）
- phase2_stop_loss = -2%（盘中止损 — 主板波动日内-2%很常见，容易被震出）
- phase3_stop_loss = -1.5%（收盘止损 — 过紧）

**调优方向:**
- phase2_take_profit: 3% → 5%（给盈利更多空间）
- phase2_stop_loss: -2% → -3%（容忍更大波动，减少被震出局）
- phase3_stop_loss: -1.5% → -2.5%（同上）
- 新增：连板持有逻辑 — 如果当日涨停且次日仍在强势区间，允许继续持有

**验收标准:**
- [ ] 卖出引擎参数按上述调整
- [ ] 回测验证调整后收益和回撤变化
- [ ] 涨停后的持有/卖出逻辑更智能（不是一律持有到收盘）

### 需求 R4: 仓位管理与风控（P1 - 重要）

**用户故事:** 作为集中持仓 1-2 只的投资者，我需要系统控制单次最大亏损和整体回撤，避免爆仓。

**当前问题:**
- 没有仓位管理逻辑 — 只推荐股票不管买多少
- 没有整体回撤控制
- 没有连续亏损后的降仓/停手机制

**需要新增:**
- 单只股票最大仓位 60%（即使只选 1 只也保留 40% 现金）
- 选 2 只时：第 1 名 50%，第 2 名 30%，留 20% 现金
- 连续亏损 3 次后自动降仓 50%（减少暴露）
- 账户总回撤 >15% 时暂停交易 3 天（冷静期）
- 单日最大亏损 >5% 触发预警

**验收标准:**
- [ ] 新增仓位管理模块（position_manager.py）
- [ ] 集成到 T1 策略的买入流程中
- [ ] 连续亏损和回撤控制逻辑正确
- [ ] 仓位参数可配置

### 需求 R5: 多策略共振信号（P1 - 重要）

**用户故事:** 作为追求确定性的投资者，我希望多个策略同时看好一只股票时才买入，提高胜率。

**当前问题:**
- 信号聚合器（signal_aggregator.py）明确排除了 T1 策略
- 5 个量化策略独立运行，与 T1 选股没有交叉验证
- 缺少"共振"作为加分项的机制

**调优方向:**
- T1 评分结果 + 趋势跟踪/动量/突破策略信号 联合验证
- 当 T1 选出的 Top 候选同时被 ≥2 个其他策略看好 → 加权提升排名
- 共振加分：+10 分（2 个策略共振）/ +15 分（3 个以上）
- 无共振的股票不一定排除，但共振股优先

**验收标准:**
- [ ] T1 选股结果与其他策略信号交叉验证
- [ ] 共振加分逻辑正确（不会导致总分超 100+15 的上限）
- [ ] 回测验证共振选股 vs 纯 T1 选股的胜率差异

### 需求 R6: 全自动化工作流（P2 - 需要）

**用户故事:** 作为没时间盯盘的上班族，我需要系统每天收盘后自动扫描、评分、推送候选，第二天早盘前看一眼就能下单。

**当前状态:**
- 有定时任务调度器（scheduler）
- 有手动触发扫描的 API
- 缺少完整的"收盘自动扫描 → 推送通知"链路

**需要完善:**
- 每个交易日 15:30 自动触发全市场扫描
- 扫描完成后生成"每日报告"— 包含 Top 1-2 候选、评分分解、策略共振状态
- 报告推送（先实现日志/API 可查，后续可对接微信/钉钉）
- 次日 9:15 生成"今日操作指令"— 包含买入标的+建议价格区间+仓位

**验收标准:**
- [ ] 定时任务在交易日 15:30 自动运行扫描
- [ ] 扫描结果包含仓位建议和策略共振信息
- [ ] 通过 API 或前端可查看每日候选报告
- [ ] 非交易日不触发扫描

## 4. 技术约束

- **数据源:** Tushare 积分账号（资金流向、北向资金等高级数据是否可用取决于积分等级）
- **交易权限:** 仅沪深主板（60xxxx）+ 深主板/中小板（00xxxx/002xxx），不可买 300/688/8/4/ST
- **涨跌停:** 主板涨跌停 ±10%（不涉及 20% 的创业板/科创板）
- **交易成本:** 佣金万 2.5 + 印花税千 1（卖出），需纳入回测计算
- **T+1 限制:** A 股当日买入不可当日卖出，策略已遵循此规则
- **最小买入单位:** 100 股/手，小资金需考虑最低买入金额限制
- **Docker 部署:** 参数优先通过环境变量/配置覆盖，不硬编码

## 5. 排除项

- **不做实盘自动下单** — 本阶段仅做选股推荐+模拟盘验证，实盘需人工下单
- **不做分钟级/Tick 级数据** — 继续使用日线 OHLCV 数据
- **不做融资融券策略** — 仅现金账户做多
- **不做期货/期权对冲** — 纯股票策略
- **不做 AI Agent 分析优化** — 本轮聚焦量化策略引擎本身
- **不做微信/钉钉推送** — 后续迭代
