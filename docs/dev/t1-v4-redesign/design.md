# T1 v4 策略设计文档：从技术指标共振到多维度评分选股

## 1. 失败原因深度分析

### 1.1 技术指标共振的本质缺陷

v3 的 4 个子策略（突破新高、均线多头、回踩反弹、缩量放量）本质上都是**价格和成交量的衍生指标**，存在三个致命问题：

1. **同源相关性**：4 个子策略的输入都是 OHLCV，它们之间高度相关。"共振"并不意味着多维度确认，而是同一信息的重复计算。当价格放量上涨时，突破新高、均线多头、缩量放量三个策略很容易同时触发——但这只是"涨了"这一个事实的三种表述。

2. **缺乏因果推断**：策略无法区分"主力建仓拉升"和"散户追涨接盘"。两者在技术形态上可能完全一致（放量突破新高），但次日走势截然不同。

3. **卖出引擎的模拟偏差**：`simulate_morning_sell()` 用日线数据近似分时走势（第 167-231 行），`sell_price = (open * 2 + high + low) / 4` 这个加权均价在实际交易中不可达，高估了卖出价格。

### 1.2 过拟合的根源

```
500 股 G12：63 笔交易，66.7% 胜率
2000 股最终：217 笔交易，45.6% 胜率（-21.1pp）
```

根源在于：
- **参数空间爆炸**：v3 有 16+ 个可调参数（RSI 阈值、涨幅上限、量比下限、上影线阈值等），在 63 笔交易上优化 16 个参数，自由度远超样本量
- **硬阈值脆弱性**：`上影线 < 1%`、`前日涨幅 < 3%` 这类硬切割在边界处不稳定。1.01% 和 0.99% 的上影线本质无差别，但一个被过滤一个通过
- **缺乏交叉验证**：G 轮优化在同一时间段的同一样本上训练和评估，没有 out-of-sample 验证

### 1.3 缺失的关键维度

| 维度 | v3 现状 | 缺失信息 | 对隔夜策略的价值 |
|------|---------|----------|-----------------|
| 资金面 | 无 | 主力净流入、北向资金 | 判断拉升是否有真实资金支撑 |
| 基本面 | 无 | ROE、净利润增速、PE | 过滤垃圾股，避免踩雷 |
| 板块面 | 无 | 行业排名、板块涨停数 | 顺势而为，做热门板块 |
| 微观结构 | 无 | 换手率、大单占比 | 区分主力行为和散户行为 |

### 1.4 卖出引擎的问题

当前卖出引擎（`t1_sell_engine.py`）的 `open_sell_threshold = 2%` 过于激进：
- 开盘涨 2% 就卖出，放弃了大部分利润空间
- 但开盘不到 2% 时又持有到 30 分钟超时，用加权均价卖出
- 这导致"赚小钱、亏大钱"的不对称结构

## 2. 新策略框架设计：T1 v4 多维度评分模型

### 2.1 核心理念转变

```
v3 思路：技术形态匹配 → 共振确认 → 买入
v4 思路：多维度评分 → 加权排序 → 选 Top-N 买入
```

v4 不再追求"精确匹配某种技术形态"，而是对每只股票计算一个综合评分，选择评分最高的 N 只买入。这从根本上避免了硬阈值过拟合问题。

### 2.2 评分模型架构

```
                    ┌─────────────────────────────────┐
                    │         T1 v4 评分模型            │
                    └─────────────┬───────────────────┘
                                  │
            ┌─────────┬───────────┼───────────┬─────────┐
            ▼         ▼           ▼           ▼         ▼
      ┌──────────┐┌──────────┐┌──────────┐┌──────────┐┌──────────┐
      │ 技术面   ││ 资金面   ││ 基本面   ││ 板块面   ││ 市场面   │
      │ 30 分    ││ 25 分    ││ 15 分    ││ 15 分    ││ 15 分    │
      └──────────┘└──────────┘└──────────┘└──────────┘└──────────┘
            │         │           │           │         │
            └─────────┴───────────┼───────────┴─────────┘
                                  ▼
                         加权总分（0-100）
                                  │
                          ┌───────┴───────┐
                          ▼               ▼
                    排序选 Top-5     一票否决过滤
                    （每日最多5只）  （ST/停牌/涨停）
```

### 2.3 各维度评分细则

#### 2.3.1 技术面评分（30 分）

不再使用硬阈值，改用连续评分函数：

| 子项 | 分值 | 计算方式 | 数据来源 |
|------|------|----------|----------|
| 趋势强度 | 10 分 | 价格在 MA5/10/20 上方的程度（连续值） | 日线 OHLCV |
| 动量质量 | 8 分 | 涨幅 2-5% 得满分，<1% 或 >7% 递减 | 日线 close |
| 量价配合 | 7 分 | 量比 1.5-3.0 得满分，sigmoid 衰减 | 日线 volume |
| K 线形态 | 5 分 | 实体占比高 + 上影线短 → 高分 | 日线 OHLC |

关键改进：用 sigmoid/线性插值替代硬阈值，消除边界不稳定性。

```python
# v3 硬阈值（脆弱）
if upper_shadow_pct > 1.0: return HOLD

# v4 连续评分（稳健）
shadow_score = 5.0 * max(0, 1 - upper_shadow_pct / 2.0)  # 0%→5分, 2%→0分
```

#### 2.3.2 资金面评分（25 分）-- 新增核心维度

| 子项 | 分值 | 计算方式 | 数据来源 |
|------|------|----------|----------|
| 主力净流入 | 10 分 | 当日大单净流入占比 > 3% 满分 | Tushare `moneyflow` |
| 换手率适中 | 5 分 | 3-8% 最优，过低无活力过高出货 | Tushare `daily_basic` |
| 连续资金流入 | 5 分 | 近 3 日主力净流入为正的天数 | Tushare `moneyflow` |
| 北向资金 | 5 分 | 当日北向净买入为正 +3，连续 3 日 +5 | Tushare `hsgt_top10` 或 AKShare |

这是 v4 最关键的新增维度。资金面直接反映了"聪明钱"的态度，是区分主力拉升和散户追涨的核心信号。

#### 2.3.3 基本面评分（15 分）-- 底线过滤

| 子项 | 分值 | 计算方式 | 数据来源 |
|------|------|----------|----------|
| ROE | 5 分 | ROE > 10% 满分，5-10% 线性 | Tushare `fina_indicator` |
| 净利润增速 | 5 分 | 同比增速 > 20% 满分 | Tushare `fina_indicator` |
| PE 合理性 | 5 分 | 行业 PE 中位数 0.5-1.5 倍范围内满分 | Tushare `daily_basic` |

基本面不用于精确选股，而是作为底线过滤：排除亏损股、高估值泡沫股、财务异常股。数据按季度更新，缓存即可。

#### 2.3.4 板块面评分（15 分）-- 顺势而为

| 子项 | 分值 | 计算方式 | 数据来源 |
|------|------|----------|----------|
| 行业当日涨幅排名 | 8 分 | 前 20% 行业满分 | AKShare `stock_board_industry_name_em` |
| 板块内涨停数 | 4 分 | >= 3 只涨停满分 | AKShare |
| 板块连续强势天数 | 3 分 | 连续 2 日排名前 30% | AKShare |

隔夜策略的核心逻辑是"强者恒强"，热门板块的股票次日高开概率显著高于冷门板块。

#### 2.3.5 市场面评分（15 分）-- 环境适配

| 子项 | 分值 | 计算方式 | 数据来源 |
|------|------|----------|----------|
| 大盘趋势 | 5 分 | 沿用 v3 MarketEnvironmentFilter 评分归一化 | 指数日线 |
| 市场情绪 | 5 分 | 涨跌比 > 1.5 满分，< 0.8 零分 | AKShare |
| 成交额活跃度 | 5 分 | 两市成交额 > 万亿满分 | AKShare |

### 2.4 一票否决条件（硬过滤）

以下条件直接排除，不参与评分：

```python
VETO_CONDITIONS = [
    "ST 或 *ST 股票",
    "上市不足 60 天",
    "科创板/北交所",
    "当日涨停（无法买入）",
    "当日跌停（趋势极弱）",
    "停牌",
    "近 5 日有涨跌停（波动过大）",
]
```

### 2.5 选股流程

```
每日 14:45 执行：
1. 获取全市场股票池（~4500 只）
2. 一票否决过滤 → 剩余 ~3000 只
3. 基本面底线过滤（ROE > 0, PE > 0）→ 剩余 ~2000 只
4. 对剩余股票计算 5 维评分
5. 按总分降序排列
6. 选 Top-5（可配置）作为买入候选
7. 检查市场面评分 >= 8 分（大盘环境安全阈值）
8. 执行买入
```

### 2.6 改进的卖出引擎

```
次日 9:25 开始：
阶段 1（9:25 集合竞价）：
  - 开盘涨 >= 5% → 卖出（大幅高开锁利）
  - 开盘跌 >= 2% → 卖出（低开止损）

阶段 2（9:30-9:45）：
  - 盘中涨 >= 3% → 挂单止盈
  - 盘中跌 >= 2% → 止损

阶段 3（9:45-10:00）：
  - 收益 > 0 → 卖出锁利
  - 收益 < -1.5% → 止损
  - 其他 → 继续观察

阶段 4（10:00 兜底）：
  - 无条件卖出
```

关键改进：
- 取消 `open_sell_threshold = 2%` 的过早止盈
- 给盈利交易更多时间发展
- 低开时快速止损，减少亏损幅度

## 3. 数据需求与获取方案

### 3.1 数据源映射

| 数据 | 首选源 | 备选源 | 频率 | 延迟 |
|------|--------|--------|------|------|
| 日线 OHLCV | 本地缓存 | Tushare/AKShare | 日 | T+0 |
| 主力资金流 | Tushare `moneyflow` | AKShare `stock_individual_fund_flow` | 日 | T+0 |
| 换手率/PE/PB | Tushare `daily_basic` | AKShare | 日 | T+0 |
| 北向资金 | Tushare `hsgt_top10` | AKShare `stock_hsgt_north_net_flow_in_em` | 日 | T+0 |
| 财务指标 | Tushare `fina_indicator` | AKShare | 季 | 季报后 |
| 行业板块 | AKShare `stock_board_industry_name_em` | Tushare | 日 | T+0 |
| 指数日线 | Tushare `index_daily` | AKShare | 日 | T+0 |

### 3.2 数据缓存策略

```
日频数据：Redis 缓存，TTL = 4 小时（盘后更新）
季频数据：PostgreSQL 存储，每季度更新一次
板块数据：Redis 缓存，TTL = 1 小时
```

### 3.3 Tushare 接口调用预算

Tushare 积分限制：每分钟 200 次调用。

```
每日运行预算：
- daily_basic（全市场）：1 次批量调用
- moneyflow（全市场）：1 次批量调用
- index_daily：1 次
- 总计：~5 次 API 调用（批量接口）
```

## 4. 防过拟合设计

### 4.1 连续评分替代硬阈值

所有评分子项使用连续函数（sigmoid、线性插值、分段线性），不使用 if-else 硬切割。

### 4.2 滚动窗口验证

回测时使用 walk-forward 方法：
```
训练窗口：6 个月
测试窗口：1 个月
滚动步长：1 个月

例：
  训练 2025.02-2025.07 → 测试 2025.08
  训练 2025.03-2025.08 → 测试 2025.09
  ...
```

### 4.3 参数稳定性测试

对每个评分权重做 +/-20% 扰动测试，要求胜率变化 < 3pp。

### 4.4 多样本验证

```
验证层级：
1. 开发集：500 只随机股票（调参用）
2. 验证集：1000 只不同股票（验证用）
3. 测试集：全市场 4000+ 只（最终评估）
```

## 5. 目标指标

| 指标 | v3 实际（2000 股） | v4 目标 | 说明 |
|------|-------------------|---------|------|
| 胜率 | 45.6% | >= 55% | 大样本稳定胜率 |
| 夏普比率 | < 1.0 | >= 2.0 | 风险调整收益 |
| 最大回撤 | 未知 | <= -8% | 控制极端风险 |
| 月度胜率波动 | 25%-70% | 标准差 < 10pp | 稳定性 |
| 盈亏比 | < 1.0 | >= 1.3 | 赚多亏少 |
| 日均交易 | ~0.9 笔 | 2-5 笔 | 足够的交易频率 |

## 7. 风险评估

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| Tushare 资金流数据延迟 | 中 | 资金面评分失效 | 降级到纯技术面 + 板块面 |
| 基本面数据季度更新太慢 | 低 | 基本面评分滞后 | 权重仅 15%，影响有限 |
| 板块轮动太快 | 中 | 板块面评分失效 | 缩短板块评估窗口到 1-2 天 |
| 极端行情（千股跌停） | 低 | 全面亏损 | 市场面评分 < 5 分时停止交易 |
| 新策略仍然过拟合 | 中 | 实盘表现不及回测 | walk-forward + 多样本验证 |
